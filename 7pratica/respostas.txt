	a. O Brackets possui uma interface mais amigável para o usuário (principalmente aquele que já esta acostumado com ambientes de desenvolvimento semelhantes: com abas, parte lateral, etc), na medida em que permite editar documentos utilizando o acessório 'mouse' e não necessita de alternar entre modos (como no vim).
	b. Para começar e codificar em C/C++/Python, o editor Brackets é a melhor escolha, na medida em que o mínimo de conhecimento necessário para editar um arquivo com ele é reduzido. Além disso, este pode ser adquirido por meio de intuição.
	c. Com o fim de produzir textos, o maior conjunto de ferramentas se encontra no Vim, contudo o Brackets possibilita recursos visuais indisponíveis no Vim (como escolha de cor no CSS ou encontrar declaração de função por seu nome).
	d. Tanto o Brackets quanto o Vim apresentam ampla disponibilidade para diversas plataformas, no entanto se observou alguns problemas de compatibilidade no Brackets ao tentar utilizá-lo em determinadas máquinas atuais.
	e. O Brackets pode ser utilizado na web com sites como Phoenix e Turbo NET. Contudo por intermédio de ambientes web como o Gitpod, o Vim tambem é acessível na web.
